WSL安装

WSL 安装

WSL安装准备工作

WSL适用于Win10 Fall Creator Update和更高版本（Win10 16215版本及以上），可以通过以下方式检查自己安装的Windows版本：

设置->系统->关于->详细信息

确保Windows系统符合安装条件后，我们需要前往主板BIOS界面，开启虚拟化支持（具体信息按照自己机型+开启虚拟化方式百度）。

启用WSL

接着我们需要启用WSL。

  • 使用 Win + x 快捷键，选择以管理员身份打开powershell，并输入以下代码 ：

    Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

    等待启用完成，重启计算机。

  • 或者我们搜索Turn Windows features on or off，打开图形界面，将Windows Subsystem for Linux 这一选项启用，等待提示完成后重启电脑。:

安装Linux分发版

若要下载并安装首选的分发版，可以选择三种做法：

  • 从 Microsoft Store 下载并安装（参阅下文）
  • 从命令行/脚本下载并安装（阅读手动安装说明）
  • 下载并手动解压缩和安装（适用于 Windows Server - 参阅此处的说明）

我们选择从Microsoft Store进行安装，便于以后的管理：

疑难解答

  • 安装失败并出现错误 0x80070003

    ** 适用于 Linux 的 Windows 子系统只能在系统驱动器（通常是 C: 驱动器）中运行。 请确保将分发版存储在系统驱动器上：

    ** 打开“设置”->“存储”->“更多存储设置: 更改新内容的保存位置”

开启WSL 2

在前面的介绍中，我们已经知道WSL2提高了文件系统性能并增加系统调用的完全兼容性，每个Linux发行版都可以使用WSL1或WSL2运行，并随时进行切换。若要开启WSL2需要进行如下设置。

WSL2 仅适用于Windows 10版本18917或更高版本

  • 确保WSL已经成功安装

  • 确保使用的是版本18917或更高版本，请加入Windows 预览体验计划，并选择 "快速" 环或 "慢速" 环，随后检查系统更新。

启用“虚拟机平台”可选组件

首先确保BIOS中已经开启CPU虚拟化，然后使用管理员身份运行Powershell输入以下代码开启虚拟机平台组件：

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

根据命令提示重新启动计算机。

使用命令行设置要由 WSL 2 支持的发行版

若要设置发行版，请运行：

wsl --set-version <Distro> 2

并且确保将Distro替换为你的发行版的实际名称。 （可使用以下命令找到这些内容：wsl -l）。 可以随时更改回 WSL 1，方法是运行与上面相同的命令，但将“2”替换为“1”。

此外，如果要使 WSL 2 成为你的默认体系结构，可以通过此命令执行该操作：

wsl --set-default-version 2

这会使你安装的任何新发行版均初始化为 WSL 2 发行版。

验证使用的WSL版本

若要验证每个发行版使用的 WSL 版本，请使用以下命令（仅适用于 Windows 版本18917或更高版本）：

wsl --list --verbose 或 wsl -l -v

上面选择的发行版现在应在“version”列下显示“2”。 既已经完成，便可以随时开始使用 WSL 2 发行版了！

疑难解答

若Powershell提示下载Linux内核更新包，请单击此链接下载适用于 AMD64 计算机的最新 WSL2 Linux 内核更新包。
